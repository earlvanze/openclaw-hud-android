Initial M1 HUD release candidate.

- Minimal green-on-black HUD presentation for AirVision M1 and Samsung DeX.
- Added per-mode physical main screen visibility control for the phone-side HUD view while the M1 presentation is active.
- Blue Light Filter now locks to Eye Care mode, matching the ASUS AirVision app behavior.
- Added AirVision demo mode for tutorials, screenshots, and HUD fit checks without a live gateway.
- AirVision HUD per-mode profile controls for view mode, placement, safe area, software brightness, virtual distance, IPD calibration, Eye Care filtering, gestures, hotkeys, Motion Sync, Light Load, and profile reset.
- HUD display targeting can now prefer AirVision, largest external, first external, or last external display routes.
- Custom 1 and Custom 2 AirVision profiles can now be renamed.
- Active AirVision profile settings can now be copied into Custom 1 or Custom 2.
- M1 brightness-key handling can now be configured for chat scrolling, Android HUD brightness, or virtual-distance stepping.
- Added AirVision app preferences with startup view, language, version, legal note, FAQ/tutorial, registration, and support links.
- AirVision 3D Mode is now stored per profile and disabled while Light Load Mode is on.
- Light Load Mode now locks IPD adjustment to mirror the ASUS AirVision app behavior.
- AirVision firmware link now shows USB device identity and firmware protocol status.
- AirVision USB firmware-link status now includes interface and endpoint diagnostics for future hardware protocol work.
- Added a HUD display identify action for the external AirVision presentation.
- Secure OpenClaw gateway pairing and reconnect handling.
- Compact chat, notification summaries, voice controls, and captions controls.
- HUD build removes SMS and Call Log permissions for Google Play.
